# 纯手动安装说明

本说明仅适用于 `Cursor`。

如果你要安装 `Codex CLI` 版本，请改用 `../../codex一键安装技能/纯手动安装/` 下的文档和脚本。

## 一、手动复制技能
把当前目录下的 `skills` 整个复制到目标机器的 Cursor 技能目录：

- Windows：`%USERPROFILE%\.cursor\skills\`
- macOS/Linux：`~/.cursor/skills/`

复制后的结构应类似：

```text
~/.cursor/skills/brainstorming/
~/.cursor/skills/playwright/
~/.cursor/skills/code-review/
```

复制完成后，重启 Cursor。

## 二、安装 context7 MCP
如果你希望 `context7` 也能使用，有两种办法。

### 办法 A：手动改 mcp.json
模板文件在：
- `context7-mcp-templates/mcp.local.windows.json`
- `context7-mcp-templates/mcp.local.unix.json`
- `context7-mcp-templates/mcp.remote.json`

说明：

- `mcp.local.windows.json` / `mcp.local.unix.json` 走本地 launcher + bundled runtime
- `mcp.remote.json` 走远程 MCP
- 默认优先使用 bundled runtime；远程模板只作为高级手动选项保留

### 办法 B：运行这里自带的 context7 专用脚本
- Windows：双击 `一键安装context7-MCP.bat`
- macOS/Linux：运行

```bash
chmod +x install_context7_mcp.sh repair_context7_mcp.sh
./install_context7_mcp.sh
```

脚本会自动：

- 检查 `node` 是否可用
- 复制发布包内置的 `context7-local` 到 `~/.cursor/context7-local`
- 生成 `run-context7.cmd` 或 `run-context7.sh`
- 写入或修复 `mcp.json`

## 三、playwright 说明
`playwright` 技能手动复制后会被 Cursor 识别，但运行相关命令仍要求目标机器具备：

```text
node
npm
npx
```

## 四、ui-ux-pro-max 说明
`ui-ux-pro-max` 技能手动复制后会被 Cursor 识别，但运行搜索脚本仍要求目标机器具备：

```text
python3 或 python
```

如果你只想单独安装或修复这个技能，可以运行：

```text
install-ui-ux-pro-max-skill.ps1
repair-ui-ux-pro-max-skill.ps1
一键安装ui-ux-pro-max技能.bat
一键修复ui-ux-pro-max技能.bat
install_ui_ux_pro_max_skill.sh
repair_ui_ux_pro_max_skill.sh
```

## 五、验证
安装后可在 Cursor 聊天中测试：

```text
Use `code-review` to review the changed files in this workspace and list findings first.
```

```text
Use `playwright` to automate a simple browser flow and explain the run steps.
```

```text
Use context7 to resolve the official React library ID.
```

## 六、关键边界

- 技能目录：`~/.cursor/skills/`
- MCP 配置：`~/.cursor/mcp.json`
- Context7 启动器：`run-context7.cmd` / `run-context7.sh`
- 安装后重启目标：`Cursor`
