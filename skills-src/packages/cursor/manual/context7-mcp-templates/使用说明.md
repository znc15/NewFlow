# context7 手动配置说明

`context7` 不是普通技能目录，它需要作为 MCP 服务配置到 Cursor。

## mcp.json 位置
- Windows：`%USERPROFILE%\.cursor\mcp.json`
- macOS/Linux：`~/.cursor/mcp.json`

## 可选模板
- `mcp.local.windows.json`
- `mcp.local.unix.json`
- `mcp.remote.json`

## 本地命令模式
适合已经把发布包里的本地 `context7` 运行时安装到 `~/.cursor/context7-local/`，并希望直接指向本地 launcher 的情况。

### Windows
把 `mcp.local.windows.json` 里的内容合并到 `mcp.json`：

```json
{
  "mcpServers": {
    "context7": {
      "command": "C:\\Users\\YOUR_USER\\.cursor\\run-context7.cmd",
      "args": []
    }
  }
}
```

### macOS / Linux
把 `mcp.local.unix.json` 里的内容合并到 `mcp.json`：

```json
{
  "mcpServers": {
    "context7": {
      "command": "/home/YOUR_USER/.cursor/run-context7.sh",
      "args": []
    }
  }
}
```

## 远程模式
如果你有 Context7 API Key，可使用远程模板 `mcp.remote.json`：

```json
{
  "mcpServers": {
    "context7": {
      "url": "https://mcp.context7.com/mcp",
      "headers": {
        "CONTEXT7_API_KEY": "YOUR_API_KEY"
      }
    }
  }
}
```

## 注意
- 如果目标机器原本已有 `mcp.json`，请手动合并，不要直接覆盖掉其他 MCP 服务。
- 修改后重启 Cursor。
- 如果本地命令模式失败，优先检查 `node` 是否可用，以及 `run-context7.cmd` / `run-context7.sh` 是否存在。
